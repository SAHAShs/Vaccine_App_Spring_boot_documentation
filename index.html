Welcome to Vaccine App!

Hi!, the Vaccination Drive Application,a demonstration project highlighting the capabilities of a vaccination portal tailored for employees. Built using the latest technologies, including Spring Boot 3.2 and Java 17,this portal illustrates how organizations can streamline the vaccination registration process for their workforce and enables employees to seamlessly register for the upcoming vaccination drive.

Vaccine Application

VaccineAppApplication.java is the main class of the Vaccine App Spring Boot application. run this class to access the application. make sure to add all configuration required.

Controller Classes

Register Control

Handles user registration. It receives data from register.jsp, validates the input, and delegates processing to the RegisterService class.

Login Control

Manages user login functionality. It receives data from login.jsp, validates the input, and delegates processing to the RegisterService class.

AddMember Control

Responsible for adding new members. It receives data from addMember.jsp, validates the input, and delegates processing to the RegisterService class.

ForgotPass Control

Deals with the forgot password feature. It receives data from forgot.jsp, validates the input, and delegates processing to the RegisterService class.

LogOut Control

Handles user logout functionality. When the user clicks the “logout” button on any JSP page, this controller invalidates the session.

Service Classes

RegisterService

Processes user registration logic, including validation and data storage to database. Sends the user about the registration and Credentials. if already user exists message is conveyed to the user.

LoginService

Manages user login logic, including validation and authentication.
if user has reached maximum login attempts account is blocked and notifies through mail.

AddMemberService

Handles the logic for adding new members, including validation and database operations.

ForgotPassService

Manages the logic for the forgot password feature, including validation and user notification. sends the otp to the user through mail and stores it and user reenters if it matches then only user is allowed to change password.

Entity Classes

Consists of two Entities, register and member.

Register Entity
columns

Member Entity

DAO Classes

DAO (Data Access Object) classes in this project extend JpaRepository<> to leverage the built-in functionality for CRUD operations provided by Spring Data JPA. The use of JpaRepository eliminates the need to write explicit HQL (Hibernate Query Language) queries. ## Example Custom Query In certain scenarios, custom queries can be written using JPQL (Java Persistence Query Language) with the @Query annotation.
Here’s an example:

import org.springframework.data.repository.query.Param; 
public interface YourRepository extends JpaRepository<YourEntity, Long> {
 @Query("SELECT CASE WHEN r.memberCount < 4 THEN true ELSE false END FROM YourEntity r WHERE r.email = :email")
  boolean memberCount(@Param("email") String email); }

DataBase

To use the Vaccine App, you need to set up a database named Vaccine_db. You can either create the database and tables manually or let Spring Boot handle it for you by enabling DDL (Data Definition Language) auto-generation in the application.properties file.

Option 1: Manual Database and Table Creation

  1. Create a database named Vaccine_db in your database management system (e.g., MySQL, PostgreSQL). 2. Create the necessary tables. Here are the SQL statements for creating tables named member and register:

sql CREATE TABLE member ( MEMBER_ID INT PRIMARY KEY, MEMBER_NAME VARCHAR(255), GENDER VARCHAR(255), DOB DATE, ID_PROOF VARCHAR(255), ID_PROOF_VALUE VARCHAR(255), VACCINE_TYPE VARCHAR(255), DOSE INT, USER_EMAIL VARCHAR(255) );

CREATE TABLE register ( ID INT PRIMARY KEY, USER_NAME VARCHAR(255), EMAIL VARCHAR(255), MOBILE_NUMBER VARCHAR(255), DOB DATE, GENDER VARCHAR(255), PASSWORD VARCHAR(255), LOGIN_ATTEMPT INT, MEMBER_COUNT INT );

Option 2: Automatic Database and Table Creation with Spring Boot

  1. Open the src/main/resources/application.properties file.
  2. Add the following configuration to enable DDL auto-generation:
    spring.jpa.hibernate.ddl-auto=update
    With this configuration, Spring Boot will automatically create/update the database tables based on your entity classes.

Note: Ensure that the database server is running before starting the Spring Boot application.

JSP Pages

The Vaccine App utilizes several JSP (JavaServer Pages) for user interfaces and interactions. Below are details about key JSP pages in the application:

Register Page: register.jsp

The register page allows users to sign up for the Vaccine App. It collects information such as user name, email, mobile number, date of birth, gender, and password.

Login Page: login.jsp

The login page provides a user authentication interface. Users can enter their credentials (email and password) to log in to the Vaccine App.

Add Member Page: addMember.jsp

On the add member page, users can input details for adding a new member to their profile. This typically includes member name, gender, date of birth, ID proof, ID proof value, vaccine type, and dose.

Forgot Password Page: forgot.jsp

The forgot password page allows users to initiate the password recovery process. Users may provide their registered email and otp, and the application will handle the rest.

Logout Button: logout.jsp (Used across various pages)

The logout button, when clicked, triggers the user logout process. It invalidates the user session, ensuring a secure logout from any page within the application.
